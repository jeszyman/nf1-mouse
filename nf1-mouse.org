** Project overview
K08 Aim 2: cfDNA monitoring of tumor biology and therapy response in /in vivo/ NF1 MPNST PDX models.
- Aim 2.1: Correlate cfDNA features with tumor characteristics at terminal bleed
- Aim 2.2: Characterize cfDNA dynamics in serial, non-terminal bleeds
- Aim 2.3: Infer tumor molecular changes from cfDNA dynamics
** Experimental system
- 4 PDX lines: JH-2-002, WU-356 (Chr8q-gain); JH-2-055, WU-487 (Chr8q-WT)
- Host strain: NOD.Cg-Rag1^{null} Il2rg^{null} (NRG)
- 5 treatment arms: vehicle, mirdametinib (A, 0.3 mg/kg QDx5 days), volasertib (VS, 50 mg/kg BIDx5 days), mirdametinib + volasertib (A+VS), untreated
- Serial submandibular bleeds (50–100 μL) at baseline and days 7, 14, 21, 28
- Terminal bleed (~400 μL) and tumor harvest at day 28
- cfDNA isolated with MagMax kit, profiled by EM-seq

** NF1 MPNST PDX model system
NF1 (neurofibromatosis type 1) affects ~1 in 2,500 individuals worldwide. Plexiform neurofibromas (PNs) can progress through atypical neurofibroma (AN) to malignant peripheral nerve sheath tumor (MPNST), the leading cause of cancer death in NF1. Lifetime MPNST risk is 8–13%; 5-year disease-specific survival is ~30%. Molecular progression follows NF1 biallelic loss → CDKN2A/B deletion → TP53 and PRC2 (SUZ12/EED) loss, with MPNST subtypes distinguished by H3K27me3-loss vs H3K27me3-retained status. Patient-derived xenograft (PDX) models recapitulate tumor histology and molecular features, enabling controlled /in vivo/ studies of cfDNA dynamics.

- PDX lines:
  | Line     | Chr8q Status | Source Institution |
  |----------+--------------+--------------------|
  | JH-2-002 | Gain         | Johns Hopkins      |
  | WU-356   | Gain         | WashU              |
  | JH-2-055 | WT           | Johns Hopkins      |
  | WU-487   | WT           | WashU              |
- Host strain: NOD.Cg-Rag1^{null} Il2rg^{null} (NRG), immune deficient
- Tumors implanted subcutaneously, early passage (P2–P4)
- Treatment begins when tumors reach 150–200 mm³
- 3 arms: vehicle, MEKi (mirdametinib), PLK1i (volasertib)
- n = 4 mice per line per arm (12 conditions, 48 mice total)
** cfDNA as translational platform
cfDNA from PDX mouse plasma provides a real-time, systemic measure of tumor biology:
- Serial submandibular bleeds enable longitudinal monitoring without sacrificing the animal
- Human-specific reads recovered after Disambiguate filtering
- Multi-omic cfDNA features: copy number (ichorCNA), fragmentomics (NNMF signatures), methylation (EM-seq), nucleosome positioning (Griffin)
- Terminal bleeds paired with matched tumor tissue for validation
- Validation chain: terminal bleed validated against final tissue state → final nonterminal bleed validated against terminal bleed → initial nonterminal bleed validated against original PDX state
- Tissue-level orthogonal assays: ATAC-seq and RNA-seq on matched tumor, with possible CUT&RUN follow-up for transcription factor binding
- [[id:c6157337-5f4a-4216-abc8-93d1a84cdbb5][PNST MRD]] and [[id:771f19d4-cd99-4013-b5de-5ba5be6b6d5d][NF1 CAPP-seq]] are related human cfDNA projects using overlapping analytical approaches
- [[id:57a06d7a-302b-488f-94af-c41a98684555][NF1-associated tissue of origin]] uses cfDNA tissue-of-origin inference across NCI, Hopkins, Mayo, WashU, and U of M samples

* Methods
** cfDNA isolation
cfDNA is isolated from mouse plasma using the MagMAX Cell-Free DNA Isolation Kit (Thermo Fisher, MAN0015629) following the manual protocol option 2 for small volumes. For 50 μL samples, plasma is diluted to 100 μL with PBS before isolation.

Right-sided SPRI bead selection (AMPure XP, 1:1 bead:sample ratio followed by 7:1 capture) is applied to remove contaminating genomic DNA when hemolysis is suspected.

**** Protocol

1. Ensure inputs - see Table ref:tab_inputs.

2. Input pre-processing - Perform pre-processing immediately prior to the input's isolation on a per-input basis. Yield calculation will require set volumes, so use the exact volumes indicated as inputs.

   - Pool mouse 1 and mouse 2 terminal bleed plasma.
   - Aliquot pooled PDX terminal plasma (200 μL, 200 μL, 50 μL, 50 μL).
   - Aliquot mouse 3 terminal bleed plasma (200 μL, 50 μL).
   - Aliquot mouse 4 terminal bleed plasma (200 μL, 50 μL).

3. Isolate cfDNA - for each isolation, follow the steps of the \citetitle{magmax_manual} \cite{magmax_manual}. (This is the manual specifically for small volume isolations). Use manual protocol, option 2. For 50 μL samples, first dilute sample volume up to 100 μL with PBS.

4. Perform right-sided bead selection as described in \citeauthor{heider2020} \citetitle{heider2020} \cite{heider2020}, methods section "Size Selection and Library Preparation of Blood-Spot Cell-Free DNA" through the 7:1 bead ratio elution:

/We opted to perform size selection to remove contaminating long gDNA fragments that would have prohibited library preparation. Thus, a right-side size selection was performed on DNA eluates using AMPure XP beads (Beckman Coulter). For this purpose, we adapted a protocol for a right-side size selection (13). Following optimization of bead:sample ratios for cfDNA fragment sizes, we used a bead:sample ratio of 1:1 to remove contaminating gDNA. The supernatant was retained as part of the right-side size-selection protocol. A second size-selection step used a 7:1 bead:sample ratio to capture remaining fragments, and the size-selected DNA was eluted in 25 μL water./

The resulting 25 μL volume is appropriate to carry forward for EM-seq library preparation \cite{NEB2024}.

--Per-sample STOPPING POINT--

Analysis for each final isolate:
   1. Qubit High sensitivity dsDNA fluorometric quantification as described in the kit user guide \cite{qubit}. Generate one standard curve for each session (can span multiple or all samples). Perform for each analyte *in nonconsecutive duplicate*. One microliter volumes are sufficient for testing. If resulting concentration is <200 pg, re-measure with 3 μL volumes. Retain concentration values and raw flourescence values including standard curve raw fluorescence.
   2. Nanodrop spectrophotometry as described in the user guide \cite{nanodrop}. Use same volume as in the Qubit and again test all samples in nonconsecutive duplicate. Use the dsDNA measurement program and retain ng/uL, A260/A280, A260/A230, and UV spectrum. Also take a A414 reading.
   3. Bioanalyzer electrophoresis with HS kit protocol. Perfrom once per analyte. Retain peak table XML and electropherogram XML.

** Methylation-aware Xengsort for EM-seq PDX read classification [#Y]
:PROPERTIES:
:ID:       A1835C32-1FEB-4902-8127-DCF343353E83
:END:
Xengsort uses k-mer hashing to classify reads as human or mouse before alignment. It was designed for unconverted WGS/WES/RNA-seq data. For EM-seq, enzymatic C-to-T conversion at unmethylated cytosines breaks k-mer identity, pushing reads into "ambiguous" or "neither" bins.

Most k-mers (those without CpGs) are unaffected by conversion and may carry enough signal for correct classification. However, CpG-dense regions (islands, shores) — which are the most informative for methylation analysis — are disproportionately affected.

Potential approach: for each k-mer in the Xengsort index that contains CpG sites, enumerate the methylation conversion variants (C or T at each CpG position) and insert all variants with the same species label. A 25-mer with N CpGs produces 2^N variants. For most k-mers (0-1 CpGs) this is 1-2 entries; for CpG-dense k-mers (2-3 CpGs) it's 4-8 entries. Hash table expansion is modest since CpG-dense regions are a small fraction of the genome.

Key questions:
- How much does the hash table expand with full CpG enumeration?
- At what CpG density per k-mer does the combinatorial expansion become impractical?
- Can a partial enumeration (e.g., cap at 2 CpGs per k-mer, dump higher to ambiguous) balance precision and compute?
- Does this outperform Disambiguate on real EM-seq data, or does Disambiguate's alignment-score comparison remain superior?

For the current PDX read handling evaluation, Xengsort is dropped in favor of Disambiguate (alignment-score comparison handles conversion naturally via bwa-meth). This idea is for potential future tool development.
